## Project

Sigil (`sigild`) is a self-tuning intelligence layer for software engineers. It runs as a background daemon observing workflow signals (file edits, terminal commands, git activity, process state, window focus), stores everything locally in SQLite, detects patterns via heuristics + optional LLM, and surfaces suggestions as desktop notifications. `sigilctl` is the CLI client.

**Repo:** `wambozi/sigil` — Go 1.24, pure Go SQLite (no CGo).

## Build & Test

```bash
make build                    # build sigild + sigilctl
make check                    # fmt + vet + test (use before committing)
make run                      # build & run with dev.toml (watches ~/workspace)
make generate                 # re-generate mocks via mockery

go test ./internal/analyzer/                          # single package
go test -run TestDetectEditThenTest ./internal/analyzer/  # single test
go vet ./...                                          # lint
```

Set `SIGIL_CLOUD_API_KEY` env var for cloud inference in dev.

## Architecture

```
Sources → Collector → Store (SQLite WAL)
                  ↓
              Analyzer (timer) → Detector (15 heuristic checks)
                  ↓               ↓ optional cloud enrichment via inference.Engine
              Notifier → notify-send / osascript
                  ↑
              Socket server ← sigilctl
                  ↑
              Actuator registry (reversible actions)
```

### Package Dependency DAG

`event` is the **leaf package** — it has zero internal imports. No package may import a package that would create a cycle. New packages must preserve this DAG:

```
event (leaf — no internal imports)
  ↑
config (no internal imports)
  ↑
store (imports event)
  ↑
inference (no internal imports — isolated backends)
  ↑
collector (imports event; defines Source interface)
  ↑
notifier (imports store interfaces)
  ↑
analyzer (imports event, store, inference, notifier)
  ↑
actuator (imports event; defines own StoreWriter interface)
  ↑
fleet (imports event, store, config)
  ↑
socket (no internal imports — pure protocol)
  ↑
cmd/sigild (wires everything together)
cmd/sigilctl (socket client + offline SQLite reads)
```

### Key Interfaces (cross-package boundaries)

| Interface | Package | Purpose |
|-----------|---------|---------|
| `Source` | collector | Event emission (File, Process, Git, Terminal, Hyprland) |
| `EventReader` | store | Read-only store queries |
| `EventWriter` | store | Write operations |
| `ReadWriter` | store | Combined read+write |
| `Backend` | inference | LLM completion (LocalBackend, CloudBackend) |
| `Platform` | notifier | OS-specific notification delivery |
| `Actuator` | actuator | Reversible action execution |
| `SuggestionStore` | notifier | Suggestion persistence |
| `RunCmd` | actuator | Shell command execution (injectable for testing) |

Consumers depend on these interfaces, not on `*store.Store` directly.

### Shared Payload Helpers

`event.IsTestOrBuildCmd`, `event.ExitCodeFromPayload`, `event.CmdFromPayload` live in `internal/event/payload.go`. Both `analyzer` and `actuator` use these — do not duplicate.

### Mocks

Generated by [mockery](https://github.com/vektra/mockery) per `.mockery.yaml`. Mocks live alongside their interfaces (`internal/<pkg>/mocks/`). Run `make generate` after changing any mocked interface.

### Data Flow

1. **Collector** fans in events from 5 sources via `Source` interface, persists to store, broadcasts on channel
2. **Analyzer** runs on timer (5m in dev, 1h default), calls `Detector.Detect()` for 15 heuristic pattern checks, optionally enriches via inference engine
3. **Notifier** surfaces suggestions per 5-level strategy (0=Silent → 4=Autonomous), rate-limits, persists all suggestions
4. **Socket server** handles ~30 JSON-over-Unix-socket methods, supports request/response + push subscriptions
5. **Actuator registry** polls registered actuators (e.g. BuildSplitActuator), executes reversible actions with undo windows

### Notification Levels

0=Silent, 1=Digest, 2=Ambient (default), 3=Conversational, 4=Autonomous (auto-executes high-confidence actions).

### Event Kinds

`file`, `process`, `hyprland`, `git`, `terminal`, `ai` — defined in `internal/event/event.go`.

## Config

TOML config at `$XDG_CONFIG_HOME/sigil/config.toml`. Sections: `daemon`, `notifier`, `inference`, `retention`, `fleet`, `schedule`, `store`. CLI flags override file values. Dev config: `dev.toml`.

## Fleet

Optional telemetry aggregation (`internal/fleet/`). Computes anonymized hourly metrics, reports adoption tier, can receive routing policy from fleet endpoint. Opt-out clears queue and disables.

## VS Code Extension

The `extensions/vscode/` directory contains a VS Code extension that surfaces sigild suggestions as IDE notification toasts.

```bash
cd extensions/vscode
npm install && npm run compile    # build
npm run package                   # produce .vsix
```

When the extension is connected, desktop notifications are automatically suppressed (single-channel routing via `HasExternalSurface` on the notifier).

## JetBrains Extension

The `extensions/jetbrains/` directory contains a Kotlin IntelliJ Platform plugin that surfaces sigild suggestions as IDE notification balloons in JetBrains IDEs (IntelliJ IDEA, PyCharm, GoLand, WebStorm).

```bash
cd extensions/jetbrains
gradle wrapper --gradle-version 8.5   # one-time: generate wrapper
./gradlew buildPlugin                 # produces .zip in build/distributions/
```

When the extension is connected, desktop notifications are automatically suppressed (single-channel routing via `HasExternalSurface` on the notifier).
